NOTES

  * The annotation "/*: function*/" in below looks like that because
    the reference implementation does not yet accept plain "function"
    as a type annotation to mean "anything callable" (ticket #153).

  * The meaning of the [[Scope]] property on 'Function' objects is
    defined elsewhere; it holds the captured scope chain of the
    'Function'.

  * The meaning of the [[Construct]] property on 'Function' objects is
    defined elsewhere; it holds executable code that constructs a new
    object instance and invokes the code of the 'Function' with the
    new object as 'this'.

  * The [[Call]] protocol of the 3rd Edition Specification has been
    abandoned in favor of a protocol based on 'meta::invoke'.  This
    simplifies the case for native objects but muddies it for callable
    host objects that do not implement 'meta::invoke'.

The class ''Function'' is a dynamic, non-final, direct subclass of ''Object'' (see ).

All objects defined by ''function'' definitions or expressions in ECMAScript are instances of the class ''Function''.

Not all objects that can be called as functions are instances of subclasses of the ''Function'' class, however. Any object that has a ''meta::invoke'' method can be called as a function. Host functions may also not be instances of ''Function'' or its subclasses, but must to some extent behave as if they are (see ). However, host functions need not provide ''meta::invoke'' methods to be callable. Any == Synopsis ==

The class ''Function'' provides the following interface: {{{ dynamic class Function extends Object { static public const length = 1 meta final function invoke( ) public const length = public var prototype = } }}}

The ''Function'' prototype object provides these direct properties: {{{ toString: function () , apply: function(thisArg, argArray) bind: function(thisArg, ...args) call: function(thisArg, ...args) }}} == Methods on the ''Function'' class object ==

new Function (p1, p2, , pn, body)

When the ''Function'' constructor is called with some arguments as part of a ''new'' expression, it creates a new ''Function'' instance whose parameter list is given by the concatenation of the //pi// arguments separated by "," and whose executable code is given by the //body// argument.

There may be no //pi// arguments, and //body// is optional too, defaulting to the empty string.

If the first character of the comma-separated concatenation of the //pi// is a left parenthesis then the list of parameters must be parseable as a //FormalParameterListopt// enclosed in parentheses and optionally followed by a colon and a return type.

Otherwise, the list of parameters must be parsable as a //FormalParameterListopt//.

If the list of parameters is not parseable as outlined in the previous two paragraphs, or if the body is not parsable as a //FunctionBody//, then a **SyntaxError** exception is thrown (see the grammar in section ).

Regardless of the form of the parameter list, it may include type annotations, default parameter values, and rest arguments. The ''Function'' constructor returns a new ''Function'' instance.

The helper function ''createFunction'' is also used by the static ''meta::invoke'' method (see ).

The magic function ''initializeFunction'' initializes the function object ''this'' from the list of parameters and the body, as specified in section . The global object is passed in as the Scope parameter.

A ''prototype'' property is automatically created for every function, to provide for the possibility that the function will be used as a constructor. It is permissible but not necessary to have one argument for each formal parameter to be specified. For example, all three of the following expressions produce the same result: {{{ new Function("a", "b", "c", "return a+b+c") new Function("a, b, c", "return a+b+c") new Function("a,b", "c", "return a+b+c") }}}

Function (p1, p2, , pn, body)

When the ''Function'' class object is called as a function it creates and initialises a new ''Function'' object. Thus the function call ''Function()'' is equivalent to the object creation expression ''new Function()'' with the same arguments. The ''Function'' class object called as a function returns a new ''Function'' instance. {{{ meta static function invoke(...args) new Function(...args); }}}

The helper function ''createFunction'' was defined along with the ''Function'' constructor (see ). === apply ( fn, thisArg=, argArray= ) === The static ''apply'' method takes arguments //fn//, //thisArg//, and //argArray//, and performs a function call using the ''meta::invoke'' method of //fn//, passing //thisArg// as the value for ''this'' and the members of //argArray// as the individual argument values.

If //fn// does not have a ''meta::invoke'' method, a **TypeError** exception is thrown. The ''apply'' method returns the value returned by //fn//.

The magic ''apply'' function performs the actual invocation (see ). === bind ( fn, thisArg=, ...args ) === The static ''bind'' method takes arguments //fn//, //thisArg//, and optionally some //args//. The ''bind'' method returns a ''Function'' object that accepts some arguments //moreargs// and which calls //fn// with //thisArg// as the ''this'' object and the values of //args// and //moreargs// as actual arguments. === call ( fn, thisArg=, ...args ) === The static ''call'' method takes arguments //fn// and //thisArg// and optionally some //args//, and performs a function call using the ''meta::invoke'' method of //fn//, passing //thisArg// as the value for ''this'' and the members of //args// as the individual argument values.

If //fn// does not have a ''meta::invoke'' method, a **TypeError** exception is thrown. The ''call'' method returns the value returned by //fn//. == Methods on Function instances == === meta::invoke ( ) === The meta method ''invoke'' is specialized to the individual function object. When called, it evaluates the executable code for the function.

The meta method ''invoke'' is typically called by the ECMAScript implementation as part of the function invocation and object construction protocols. When a function or method is invoked, the ''invoke'' method of the function or method object provides the code to run. When a function is used to construct a new object, the ''invoke'' method provides the code for the constructor function.

The signature of the meta method ''invoke'' is determined when the ''Function'' instance is created, and is determined by the text that defines the function being created. The meta method ''invoke'' is ''final''; therefore subclasses can add properties and methods but can't override the function calling behavior. (Ticket #173.) While it is necessary that the ''invoke'' method is completely magic in ''Function'' instances, it's not clear it needs to be magic for instances of subclasses of ''Function'', because these can be treated like other objects that have ''invoke'' methods (and which already work just fine). Therefore it should not be ''final''. The meta method ''invoke'' returns the value produces by the first ''return'' statement that is evaluated during the evaluation of the executable code for the function represented by this ''Function'' object. === intrinsic::toString ( ) === The intrinsic ''toString'' method converts the executable code of the function to a string representation. This representation has the syntax of a //FunctionDeclaration// or //FunctionExpression//. Note in particular that the use and placement of white space, line terminators, and semicolons within the representation string is implementation-dependent. The intrinsic ''toString'' method returns a string.

The private function ''toString'' is implementation-dependent. === intrinsic::apply ( thisObj=, args= ) === The intrinsic ''apply'' method calls the static ''apply'' method with the current value of ''this'' as the first argument. The intrinsic ''apply'' method returns the result of the static ''apply'' method. === intrinsic::bind ( thisObj=, ...args) === The intrinsic ''bind'' method calls the static ''bind'' method with the current value of ''this'' as the first argument. The intrinsic ''bind'' method returns the result of the static ''bind'' method. === intrinsic::call ( thisObj=, ...args) === The intrinsic ''call'' method calls the static ''apply'' method with the current value of ''this'' as the first argument. The intrinsic ''call'' method returns the result of the static ''call'' method. == Properties of ''Function'' instances ==

In addition to the required internal properties, every function instance has a ''meta::invoke'' property, a [[Construct]] property and a [[Scope]] property (see sections 8.6.2 and 13.2). === length ===

The value of the constant ''length'' property is the number of non-rest arguments accepted by the function.

The value of the ''length'' property is an integer that indicates the "typical" number of arguments expected by the function. However, the language permits the function to be invoked with some other number of arguments. The behaviour of a function when invoked on a number of arguments other than the number specified by its length property depends on the function. === prototype ===

The initial value of the ''prototype'' property is a fresh ''Object'' instance.

The value of the ''prototype'' property is used to initialise the internal [[Prototype]] property of a newly created object before the ''Function'' instance is invoked as a constructor for that newly created object. == Invoking the ''Function'' prototype object ==
