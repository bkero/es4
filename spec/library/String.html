= The class "String" =

There are two native string classes, ''String'' and ''string'', the latter a subclass of the former. Instances of ''String'' are compatible with ''String'' objects in 3rd Edition, while instances of ''string'' are compatible with ''string'' values in 3rd Edition. == The class ''String'' ==

The String class is dynamic, non-final, and holds a sequence of Unicode character values (its "string value"). The string value is immutable and is set at the time a String object is constructed. === Synopsis === {{{ dynamic class String { static const length: uint = 1 static const prototype: Object = } }}} Should ''string'' methods be generic (ie unconstrained in their object argument), or only ''String'' methods? The prototype methods on ''string'' have to be generic, for sure, but the static methods are new and we can choose.
