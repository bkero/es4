ECMAScript provides a primitive representation for truth values in instances of the class ''boolean''. It is primitive in the sense that the representation is directly operated upon by relational operators and control flow statements and control flow expressions, and also in the sense that the class ''boolean'' is a final and non-dynamic class for which implementations may provide efficient representations.

ECMAScript also provides the class ''Boolean'', which is a dynamic non-final class that holds ''boolean'' values. Instances of ''Boolean'' are usually converted to ''boolean'' when operated upon by relational operators and control flow statements.

The class ''Boolean''

The class ''Boolean'' is a dynamic non-final class that represents a boolean value. Instances of ''Boolean'' are usually converted to ''boolean'' when operated upon by relational operators and control flow statements. == Synopsis ==

The class ''Boolean'' provides the following interface: {{{ dynamic class Boolean { static const length: uint = 1 } }}}

The ''Boolean'' prototype object provides the following direct properties: {{{ toString: function (this: (Boolean,boolean)) valueOf: function (this: (Boolean,boolean)) }}}

The default //value// is **false**. The ''Boolean'' constructor is implementation-defined. === Boolean( value= ) === The ''Boolean'' class object called as a function converts //value// to boolean. The ''boolean'' class object called as a function returns a ''boolean'' object (not a ''Boolean'' object). == Methods on ''Boolean'' instances == === intrinsic::toString ( ) === The intrinsic ''toString'' method converts this boolean value to a string. The intrinsic ''valueOf'' method returns a ''boolean'' object (not a ''Boolean'' object). == Methods on the ''Boolean'' prototype object == The methods on the ''Boolean'' prototype object invoke their intrinsic counterparts. The methods on the ''Boolean'' prototype object return what their intrinsic counterparts return.
