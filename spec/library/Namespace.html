The class ''Namespace'' is a final, non-dynamic, direct subclass of ''Object''. It reflects a namespace as an opaque datum. Namespaces are created as a result of the evaluation of the ''namespace'' pragma, which defines a new namespace and creates a binding for it. Namespaces are reflected as ''Namespace'' objects when namespace bindings are referenced in expressions. The ''Namespace'' class is new in the 4th Edition of this Standard. == Synopsis ==

The class ''Namespace'' provides the following interface: {{{ __ES4__ final class Namespace extends Object { intrinsic function toString(): string } }}}

The ''Namespace'' prototype object provides the following direct properties: {{{ toString: function () }}} == Methods on the ''Namespace'' class object ==

There are no methods on the ''Namespace'' class object. In particular, the class ''Namespace'' does not have a publicly accessible constructor. == Methods on ''Namespace'' instances == === intrinsic::toString() === The intrinsic ''toString'' method converts the ''Namespace'' object to a string. If the ''Namespace'' object was created with an explicit name then the string contains that name as a substring. A ''Namespace'' object can be created with an explicit name by providing the name as a string in the ''namespace'' pragma. The ''toString'' method returns an implementation-defined string. The ''toString'' method is implementation-defined. == Methods on the ''Namespace'' prototype object == The methods on the ''Namespace'' prototype object delegate to their corresponding intrinsic methods. The methods on the ''Namespace'' prototype object return what their corresponding intrinsic methods return. {{{ prototype function toString(this:Namespace) this.intrinsic::toString() }}}
